# 员工民主评议评分系统调整方案

## 一、系统架构调整

### 1.1 角色管理模块

系统已支持四个公司角色的配置管理：
- 正职领导
- 中层正职
- 中层副职及助理
- 管理人员

### 1.2 评分规则引擎

开发了灵活的评分公式配置系统，支持：
- 通过可视化界面定义各角色的综合得分计算公式
- 支持不同评议主体权重设置
- 公式参数支持动态调整，不硬编码
- 保留历史修改记录

## 二、数据库表结构设计

### 2.1 评分规则表 (scoring_rules)

```sql
CREATE TABLE scoring_rules (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255) NOT NULL COMMENT '规则名称',
    target_role VARCHAR(255) NOT NULL COMMENT '目标角色',
    description TEXT COMMENT '描述',
    is_active BOOLEAN DEFAULT TRUE COMMENT '是否启用',
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL
);
```

### 2.2 评分规则权重表 (scoring_rule_weights)

```sql
CREATE TABLE scoring_rule_weights (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    scoring_rule_id BIGINT UNSIGNED NOT NULL,
    evaluator_role VARCHAR(255) NOT NULL COMMENT '评议者角色',
    weight DECIMAL(5, 2) NOT NULL COMMENT '权重百分比',
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL,
    FOREIGN KEY (scoring_rule_id) REFERENCES scoring_rules(id) ON DELETE CASCADE
);
```

### 2.3 评分规则历史表 (scoring_rule_histories)

```sql
CREATE TABLE scoring_rule_histories (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    scoring_rule_id BIGINT UNSIGNED NOT NULL,
    name VARCHAR(255) NOT NULL,
    target_role VARCHAR(255) NOT NULL,
    description TEXT,
    weights TEXT NOT NULL COMMENT '权重配置JSON',
    changed_by BIGINT UNSIGNED NOT NULL COMMENT '修改人ID',
    change_reason VARCHAR(255) COMMENT '修改原因',
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL,
    FOREIGN KEY (scoring_rule_id) REFERENCES scoring_rules(id) ON DELETE CASCADE,
    FOREIGN KEY (changed_by) REFERENCES users(id) ON DELETE CASCADE
);
```

### 2.4 评分等级表 (scoring_levels)

```sql
CREATE TABLE scoring_levels (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255) NOT NULL COMMENT '等级名称',
    min_score DECIMAL(5, 2) NOT NULL COMMENT '最低分数',
    max_score DECIMAL(5, 2) NOT NULL COMMENT '最高分数',
    color VARCHAR(255) DEFAULT '#409EFF' COMMENT '颜色标识',
    sort_order INT DEFAULT 0 COMMENT '排序',
    is_active BOOLEAN DEFAULT TRUE COMMENT '是否启用',
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL
);
```

### 2.5 问题评分等级表 (question_scoring_levels)

```sql
CREATE TABLE question_scoring_levels (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    question_id BIGINT UNSIGNED NOT NULL,
    level_name VARCHAR(255) NOT NULL COMMENT '等级名称',
    score DECIMAL(5, 2) NOT NULL COMMENT '分数',
    description TEXT COMMENT '描述',
    sort_order INT DEFAULT 0 COMMENT '排序',
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL,
    FOREIGN KEY (question_id) REFERENCES evaluation_questions(id) ON DELETE CASCADE
);
```

### 2.6 现有表结构更新

## 三、核心功能模块设计

### 3.1 评分规则管理模块

**功能描述**：
- 创建、编辑、删除评分规则
- 配置各评议角色的权重
- 查看规则修改历史
- 权重总和校验（必须等于100%）

**API接口**：
- `GET /api/scoring-rules` - 获取所有评分规则
- `POST /api/scoring-rules` - 创建评分规则
- `GET /api/scoring-rules/{id}` - 获取单个评分规则
- `PUT /api/scoring-rules/{id}` - 更新评分规则
- `DELETE /api/scoring-rules/{id}` - 删除评分规则
- `GET /api/scoring-rules/{id}/history` - 获取规则历史
- `GET /api/scoring-rules/role/{role}` - 根据角色获取规则
- `POST /api/scoring-rules/calculate` - 计算得分

### 3.2 评分等级管理模块

**功能描述**：
- 创建、编辑、删除评分等级
- 配置分数区间和等级名称
- 设置等级颜色标识
- 根据分数自动匹配等级

**API接口**：
- `GET /api/scoring-levels` - 获取所有评分等级
- `POST /api/scoring-levels` - 创建评分等级
- `PUT /api/scoring-levels/{id}` - 更新评分等级
- `DELETE /api/scoring-levels/{id}` - 删除评分等级
- `GET /api/scoring-levels/score/{score}` - 根据分数获取等级

### 3.3 评分计算服务

**核心类**：`ScoringService`

**主要方法**：
```php
// 计算维度得分
public function calculateDimensionScore($evaluateeId, $planId, $dimensionId)

// 按评议者角色计算得分
public function calculateScoreByEvaluatorRole($evaluateeId, $planId)

// 计算综合得分
public function calculateTotalScore($evaluateeId, $planId)

// 生成评估报告
public function generateReport($evaluateeId, $planId)

// 批量生成报告
public function batchGenerateReports($planId)
```

## 四、评分流程时序图

```
评估者 -> 系统: 登录系统
系统 -> 评估者: 显示评估任务列表
评估者 -> 系统: 选择评估任务
系统 -> 评估者: 显示评估表单
评估者 -> 系统: 提交评估答案
系统 -> 数据库: 保存评估答案
系统 -> 评估者: 显示提交成功

被评估者 -> 系统: 查看评估报告
系统 -> 数据库: 获取评估答案
系统 -> ScoringService: 计算综合得分
ScoringService -> 数据库: 获取评分规则
ScoringService -> ScoringService: 按角色计算均分
ScoringService -> ScoringService: 应用权重公式
ScoringService -> 数据库: 获取评分等级
ScoringService -> 系统: 返回计算结果
系统 -> 被评估者: 显示评估报告
```

## 七、系统原型

### 7.1 评分规则管理界面

**主要功能**：
- 规则列表展示
- 创建/编辑规则对话框
- 权重配置（支持动态添加/删除）
- 权重总和实时校验
- 历史记录查看

**界面元素**：
- 规则名称输入框
- 目标角色下拉选择
- 描述文本框
- 权重配置区域（评议角色 + 权重百分比）
- 权重总和显示（颜色标识是否等于100%）
- 修改原因输入框（编辑时）

### 7.2 评分等级管理界面

**主要功能**：
- 等级列表展示
- 创建/编辑等级对话框
- 分数区间配置
- 颜色标识选择
- 排序设置

**界面元素**：
- 等级名称输入框
- 最低分数输入框
- 最高分数输入框
- 颜色选择器
- 排序输入框

### 7.3 评分计算流程

1. 系统根据被评估者的角色自动匹配评分规则
2. 按评议者角色分组计算均分
3. 应用权重公式计算综合得分
4. 根据分数匹配评分等级
5. 生成详细的评估报告

## 八、系统扩展性设计

### 8.1 支持新增角色

系统设计支持动态添加新角色：
- 在用户管理中添加新角色
- 在评分规则中为新角色配置权重
- 在评估维度和问题中指定适用角色

### 8.2 支持调整评分指标及权重

系统设计支持动态调整：
- 评估维度可以动态添加、编辑、删除
- 评估问题可以动态配置权重
- 评分规则可以动态调整权重配置
- 所有修改都有历史记录

### 8.3 数据准确性保障

- 权重总和校验（必须等于100%）
- 分数区间校验（最低分数必须小于最高分数）
- 数据完整性校验（必填字段验证）
- 事务处理（确保数据一致性）

### 8.4 操作便捷性设计

- 可视化界面配置
- 实时预览计算结果
- 友好的错误提示
- 批量操作支持
- 数据导出功能

## 十、总结

本方案设计了一套完整的员工民主评议评分系统调整方案，实现了以下核心功能：

1. **动态评分规则配置**：支持通过可视化界面配置各角色的综合得分计算公式，权重参数支持动态调整。
2. **灵活的评分等级管理**：支持自定义评分等级和分数区间，根据分数自动匹配等级。
3. **完整的评分计算服务**：实现了按角色计算均分、应用权重公式、生成评估报告等功能。
4. **历史记录追踪**：所有评分规则的修改都有历史记录，便于追溯和审计。
5. **良好的扩展性**：支持未来新增角色、调整评分指标及权重，保证系统的可持续发展。

系统设计充分考虑了数据准确性、操作便捷性和系统扩展性，能够满足公司多角色、多维度的综合评分需求。
